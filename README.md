## ğŸš€ Quick Start

### Prerequisites
- Node.js (v16 or higher)
- MongoDB (local or MongoDB Atlas)
- Google AI Studio API Key ([Get one here](https://makersuite.google.com/app/apikey))

### Installation

1. **Clone the repository**
   ```bash
   git clone <repository-url>
   cd "Hackathon EISD"
   ```

2. **Install all dependencies**
   ```bash
   npm run install:all
   ```

3. **Configure Environment Variables**

   Create a `.env` file in the `server` directory:
   ```bash
   cd server
   copy .env.example .env
   ```

   Edit `.env` and add your configuration:
   ```env
   PORT=5000
   NODE_ENV=development
   
   # MongoDB Connection
   MONGODB_URI=mongodb://localhost:27017/eduai
   # Or use MongoDB Atlas:
   # MONGODB_URI=mongodb+srv://<username>:<password>@cluster.mongodb.net/eduai
   
   # Google AI Studio (Gemini API) - REQUIRED for AI features
   GEMINI_API_KEY=your_gemini_api_key_here
   
   # CORS
   CLIENT_URL=http://localhost:5173
   ```

4. **Start MongoDB** (if using local MongoDB)
   ```bash
   mongod
   ```

5. **Run the application**
   ```bash
   # From root directory
   npm run dev
   ```

   This will start:
   - Frontend: http://localhost:5173
   - Backend: http://localhost:5000

## ğŸ” Security - Gemini API Key

âš ï¸ **IMPORTANT**: The Gemini API key is stored ONLY in the backend `.env` file and is NEVER exposed to the frontend.

### How it works:
1. API key is stored in `server/.env`
2. Backend reads key using `process.env.GEMINI_API_KEY`
3. Only backend makes requests to Gemini API
4. Frontend receives processed results via REST API

### Best Practices:
- âœ… Never commit `.env` to version control
- âœ… Add `.env` to `.gitignore`
- âœ… Use environment variables in production
- âœ… Restrict API key domains in Google AI Studio
